# Проверка версии 4 в Max

Автоматические тесты проверяют JS и граф patchcord. Этот список нужен для
фактического поведения `bach` 0.8.2.0 beta/0.8.3.

## 1. Открытие

1. Открыть `max/WebernPersona.maxpat` из новой папки целиком.
2. Убедиться, что Max не находит старые одноимённые файлы через Search Path.
3. Должны быть видны три панели, Persona и пустой десятистрочный score.
4. В Console не должно быть ошибок JS/JSON или неизвестных атрибутов.

## 2. Новый ряд

1. Поставить `Seed = 1913`, `Events = 36`.
2. Нажать **Generate Material** три раза.
3. Каждый раз ROW должен содержать 12 разных классов высоты и меняться.
4. Изменить Seed на другое число, затем снова ввести `1913`.
5. Первое Generate после возврата должно повторить первый ряд шага 2.

Seed задаёт последовательность, но не замораживает одну перестановку.

## 3. Build всегда пересобирает

1. Нажать **Build Score** без предварительного Generate.
2. Должны последовать `generated`, `quantize_requested`,
   `score_data_received`, затем
   `score_ready dynamics … articulations … annotations … tempi …`.
3. Перерисовать обе кривые и снова нажать Build: форма должна измениться.
4. Ничего не менять и нажать Build ещё раз: появится новый ROW и новый score.
5. Не должно быть `bach.score: unknown message: D`; это ошибка старой v3.0.0.

## 4. Пять характеров

При выборе профиля должны сразу меняться BPM и обе видимые линии.

| Профиль | Что проверить в нескольких генерациях |
|---|---|
| I | BPM 50, много воздуха, pppp/pp по умолчанию, сурдинные краски, `zögernd–a tempo` |
| II | BPM 100, длиннее микро-фразы, больше общих атак к концу, `drängend`, `poco rit.`, `rasch`, широкий подъём |
| III | BPM 40, несколько инструментов на общем E, tremolo/trill, долгие перекрытия, `kaum hörbar`, `verklingend` |
| IV | BPM 60, жесты по три звука, ответ/эхо, фоновые продления, `wie ein Hauch`, `sehr gebunden` |
| V | BPM 152, плотная вершина, затем `ruhig` и редкий тихий хвост, `molto rit.` |

Профиль вероятностный: одна конкретная техника не обязана появиться в каждом
коротком эскизе, но общая фактура и темп обязаны различаться.

## 5. Регистр и фраза

1. Просмотреть глокеншпиль, флейту, скрипку и арфу в профилях I/III/IV.
2. Обычные события должны находиться в читаемом среднем/рабочем регистре.
3. Крайняя нота допустима редко и около фактурной/динамической вершины.
4. Найти группы, где один инструмент играет 2–5 связанных событий: это
   временный focus, а не случайное повторение партии.
5. Найти `exchange`/`echo`: короткий жест должен переходить к родственной
   краске, сохраняя ритмическое/контурное родство.
6. В `trace` Max Console строки содержат `solo-line`, `exchange`, `echo`,
   `homorhythm` или `pedal-tremolo`.

## 6. Dynamics / Time

1. Поставить линию в самом низу от начала до конца — ожидается `pppp`.
2. Поставить в самом верху — ожидается `fff`.
3. Нарисовать низко → высоко → низко: ключевые отметки и вилочки должны
   следовать этой драматургии по времени.
4. Повторить в профиле I: пользовательский подъём к fff теперь разрешён;
   профиль задаёт исходную, а не принудительно ограниченную кривую.

## 7. Оформление

На score должны встречаться только контекстно оправданные:

- `staccato/staccatissimo`, accent, portato, trill, tremolo;
- `dolce`, `dolcissimo`, `espress.`, `molto espr.`, `deutlich`;
- `mit Dämpfer`, `Flatterzunge`, `col legno`, `mit Schwammschlägel`;
- `wie ein Hauch`, `kaum hörbar`, `verklingend`;
- изменения темпа и `zögernd/a tempo/drängend/rit./rasch/ruhig`.

Длинные длительности через тактовую черту должны дать нативные ties.
Фразовые slur программно не создаются: публичный API bach 0.8.2/0.8.3 их не
поддерживает. Связность показывают группировка, portato/legato-ремарки и ties.

## 8. MusicXML

1. Нажать **Export XML** и сохранить файл в `examples/`.
2. Выполнить:

```sh
node tools/check_musicxml.js examples/<имя>.musicxml
```

3. Импортировать в MuseScore/Dorico.
4. Проверить десять партий, ноты/паузы/ties, динамику, вилочки, штрихи, темпы
   и текстовые directions.

Если знак есть в score, но пропал в XML, сохранить экспорт и Max Console:
это отделяет ошибку адресации slot от ограничения экспортёра.
